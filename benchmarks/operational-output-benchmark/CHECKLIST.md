# Benchmark Checklist

Use esta checklist para cada output real.

## Antes De Avaliar

- [ ] Confirmar que a squad executou normalmente.
- [ ] Copiar o output original para `01-original-output.md`.
- [ ] Registrar squad, run id, artefato, versão e pedido original.
- [ ] Não propor nova layer antes da auditoria.

## Auditoria Hostil

- [ ] Gerar `02-hostile-review.md`.
- [ ] Procurar trechos genéricos.
- [ ] Procurar overclaim.
- [ ] Procurar sensação de IA.
- [ ] Procurar sensação de template.
- [ ] Procurar teatralidade ou founder performance.
- [ ] Registrar pontos humanos fortes.
- [ ] Registrar pontos operacionais fortes.

## Scorecard

- [ ] Preencher Human Reality Score.
- [ ] Preencher Operational Specificity.
- [ ] Preencher Genericness Risk.
- [ ] Preencher Overclaim Risk.
- [ ] Preencher Template Feeling.
- [ ] Preencher Founder Authenticity.
- [ ] Preencher Proof Strength.
- [ ] Preencher Emotional Manipulation Risk.

## Refinamento

- [ ] Refinar apenas o output.
- [ ] Não criar runtime, engine, layer ou arquitetura nova.
- [ ] Preservar o que funcionou.
- [ ] Cortar o que parece genérico, inflado ou performático.
- [ ] Comparar original vs refinado em `06-comparison.md`.

## Decisão

- [ ] O output ficou perceptivelmente mais real?
- [ ] O output exigiria menos edição humana?
- [ ] A mudança melhorou credibilidade, especificidade ou clareza operacional?
- [ ] Se não houve melhoria perceptível, registrar como hipótese descartada.
