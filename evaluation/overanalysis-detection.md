# Overanalysis Detection

## Regra

O diagnóstico deve parar quando já existir clareza suficiente para agir.

## Sinais De Overanalysis

- Muitas categorias e pouca decisão.
- Explicação que não altera prioridade.
- Detalhamento que não muda o próximo passo.
- Nova camada conceitual onde bastava uma recomendação.
- Incerteza virando investigação eterna.
- Output parece inteligente, mas o operador ainda não sabe o que fazer amanhã.

## Sinais De Simplificação Excessiva

- Ignora dependência humana crítica.
- Trata processo informal como detalhe.
- Propõe automação antes de fonte de verdade.
- Não separa fato observado de hipótese.
- Não considera maturidade operacional.

## Teste De Corte

Para cada bloco do diagnóstico, perguntar:

1. Isso muda a prioridade?
2. Isso muda o risco?
3. Isso muda o próximo passo?
4. Isso ajuda founder ou executivo a entender mais rápido?

Se a resposta for `não` para todos, cortar.
